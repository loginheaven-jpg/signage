# 디지털 게시판 시스템 (Digital Signage) v2

교회 입구/식당 등 다중 사이트에 설치된 디스플레이를 중앙에서 통합 제어하는 디지털 사이니지 시스템입니다.

## 아키텍처

```
┌─────────────────────────────────────────────────────┐
│ 클라이언트 PC (현관/식당 등)                          │
│                                                     │
│  ┌─ 드라이브 동기화 ─┐   ┌─ 편성표 (로컬 JSON) ─┐   │
│  │ 5분 주기 자동 체크  │   │ 호스트에서 수신·저장  │   │
│  │ + 호스트 푸시 즉시  │   │ 호스트 없으면 마지막  │   │
│  └────────┬──────────┘   │ 저장본으로 계속 재생  │   │
│           ▼              └──────────┬───────────┘   │
│  ┌─ 로컬 캐시 ──────┐              │               │
│  │ 이미지/동영상 파일 │              ▼               │
│  └────────┬──────────┘   ┌─ 재생 엔진 ─────────┐   │
│           └──────────────▶│ 편성표 기반 자동 순환 │   │
│                           └─────────────────────┘   │
└─────────────────────────────────────────────────────┘
         ▲                          ▲
         │ 파일 동기화               │ 편성표 푸시 + sync_now
         ▼                          │
┌─────────────────┐         ┌───────┴────────────┐
│  구글 드라이브    │         │  호스트 PC          │
│  (콘텐츠 저장소)  │         │  (편성표 편집/관리)  │
└─────────────────┘         └────────────────────┘
```

## 핵심 동작 원리

- **호스트 꺼져도 재생 계속**: 클라이언트가 구글드라이브에서 직접 파일을 동기화하고 로컬에 캐시
- **편성 변경 즉시 반영**: 호스트에서 편성표 저장 시 WebSocket으로 모든 클라이언트에 즉시 푸시
- **5분 주기 백그라운드 체크**: 호스트 없이도 드라이브 변경사항 자동 감지
- **인터넷 끊겨도 재생**: 로컬 캐시 파일로 마지막 편성표 기반 계속 재생

## 설치 및 실행

### 사전 요구사항
- Node.js 18+ (https://nodejs.org)
- 구글 드라이브 서비스 계정 키 (JSON)

### 호스트 (관리자 PC)

```bash
cd host
npm install
npm start
```
→ 브라우저에서 http://localhost:3000 접속

### 클라이언트 (디스플레이 PC)

```bash
cd client
npm install
```

`config.js` 수정:
```javascript
module.exports = {
  HOST_URL: 'ws://호스트PC의IP:3000',
  CLIENT_NAME: '현관',
  GDRIVE_FOLDER_ID: '구글드라이브폴더ID'
};
```

`credentials/service-account.json`에 서비스 계정 키 배치 후:
```bash
npx electron .
```

### 부팅 시 자동 실행 (Windows)

1. `Win+R` → `shell:startup` 입력
2. 바로가기 생성: 대상 → `cmd /c "cd /d C:\dev\signage\client && npx electron ."`

## 편성표 사용법

1. 호스트 웹 UI에서 "편성표" 섹션의 "항목 추가" 클릭
2. 좌측/우측 파일 선택 (모니터 1대인 경우 우측은 비워도 됨)
3. 표출 시간(초) 설정
4. 동영상인 경우 "원본 길이" 또는 "지정 시간" 선택
5. "저장 및 적용" 클릭 → 모든 클라이언트에 즉시 반영

## 모니터 자동 적응

| 모니터 수 | 동작 |
|---|---|
| 2대 | 좌/우 동시 표출 |
| 1대 | 좌 파일 재생 → 우 파일 순차 재생 |

## 프로젝트 구조

```
digital-signage/
├── host/                    # 호스트 서버
│   ├── server.js           # Express + WebSocket 서버
│   ├── gdrive-sync.js      # 구글 드라이브 동기화 모듈
│   ├── public/index.html   # 관리자 웹 UI
│   ├── data/schedule.json  # 편성표 저장
│   ├── uploads/            # 콘텐츠 파일 (드라이브 동기화)
│   └── credentials/        # 서비스 계정 키 (git 제외)
├── client/                  # 클라이언트 플레이어
│   ├── main.js             # Electron 메인 프로세스
│   ├── player.html         # 재생 화면 (렌더러)
│   ├── preload.js          # IPC 브릿지
│   ├── gdrive-sync.js      # 드라이브 직접 동기화
│   ├── schedule.js         # 편성표 로컬 관리
│   ├── config.js           # 클라이언트 설정
│   └── credentials/        # 서비스 계정 키 (git 제외)
└── README.md
```
